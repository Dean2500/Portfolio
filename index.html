De-identifier Project

Overview

Developed collaboratively by a team of two developers.

Project Started June 2023
Project Ended July 2023
Platform Linux
Code Java

User Authorization and Legal Compliance

The program operates under the assumption that users accessing sensitive information are legally entitled to do so. This prerequisite ensures compliance with privacy regulations and safeguards against unauthorized access or misuse of confidential data. Users must possess the necessary permissions and authorizations to view and modify sensitive information, thereby mitigating the risk of privacy breaches and legal liabilities.



Functionality

Command Line Program

The functionality of this software primarily revolves around its command-line interface, making it accessible and efficient for users who are comfortable with terminal-based interactions. Users can execute various commands to initiate the desired operations, such as identifying specific types of sensitive information within data sets or packet captures.


Data Identification and Filtering

The program is adept at recognizing sensitive information such as IP addresses, email addresses, names, and student numbers. To minimize false positives, it employs sophisticated algorithms that consider not only the presence of keywords but also the context in which they appear. This involves examining punctuation and surrounding text to ensure accurate identification.


System Awareness and Compatibility

Understanding common Linux system commands and structures is crucial for reducing errors and enhancing the program's compatibility across different environments. By leveraging knowledge of Linux systems, the software can navigate file systems, logs, and other unstructured data sources efficiently.


Anonymization and Data Preservation

An important aspect of the program is its ability to anonymize sensitive data while preserving the overall structure and integrity of the original information. This process involves replacing identifiable elements with placeholders or randomized values, ensuring compliance with privacy regulations without compromising data utility.


Client-Server Architecture

The software offers both client-server and offline modes, providing flexibility in how users interact with it. In client-server mode, users can remotely access the program's functionality and synchronize changes securely over a network connection. This architecture facilitates collaboration and centralized management of sensitive data across multiple devices or locations.


Secure Communication

To ensure data integrity and confidentiality, the program utilizes Secure Socket Layer (SSL) or Transport Layer Security (TLS) protocols for communication between clients and servers. This encryption mechanism safeguards sensitive information during transmission, mitigating the risk of interception or unauthorized access.


User Interaction and Decision-Making

In cases where the program encounters ambiguous or uncertain data, it presents these findings to the user for review and decision-making. Users are empowered to determine whether to modify or preserve the identified information, providing a layer of control and transparency in the anonymization process.


Automated Synchronization and Efficiency

To streamline the anonymization process, the program incorporates automated synchronization features specifically designed for client-server mode. In this mode, user preferences and modifications are saved and synchronized seamlessly across multiple devices over a secure connection. This minimizes the need for manual intervention and accelerates the overall deidentification process. However, in offline mode, changes are saved exclusively to a local file without synchronization across devices. This distinction ensures efficient data management while catering to different usage scenarios.



Project Outcome

Effective Identification of Sensitive Information

The program demonstrated satisfactory performance in identifying various types of sensitive information, such as IP addresses, emails, names, and student numbers, within unstructured data. It effectively balanced accuracy and false positives at acceptable levels, ensuring thorough data analysis while minimizing errors in identification.


Thorough Data Analysis

By delving into IP packet captures and employing detailed checks, the program ensured a comprehensive analysis of data, minimizing the risk of overlooking critical information.


Minimization of False Positives

Through awareness of system commands and meticulous filtering processes, the program effectively reduced false positives to maintain data accuracy and reliability.


Robust Anonymization Techniques

The program implemented robust anonymization methods while preserving the underlying data structure, ensuring compliance with privacy regulations without compromising data integrity.


Flexible Deployment Options

Offering both client-server and offline modes, the program provided users with flexibility in accessing and managing sensitive data according to their preferences and operational requirements.


Secure Communication

Utilizing Secure Socket Layer (SSL) communication, the program ensured secure transmission of data, safeguarding against unauthorized access and data breaches.


Efficient Handling of Unstructured Data

By managing unstructured data across various file systems and logs, the program demonstrated adaptability and efficiency in processing diverse data formats.


User Empowerment and Decision-Making

The program empowered users by presenting potentially ambiguous data for review and decision-making, allowing them to determine whether to modify or preserve identified information.


Streamlined Workflow

Through automated synchronization features and efficient data saving mechanisms, the program optimized workflow and minimized user intervention, enhancing overall productivity and efficiency.


Enhanced Collaboration

With synchronized changes facilitated in client-server mode, the program promoted collaboration and centralized management of sensitive data, facilitating seamless communication and decision-making processes.

USB Scanner

Overview

Project Started July 2023
Project Ended July 2023
Platform Linux
Code Python


Functionality

USBScanner Program Functionality

The USBScanner program operates seamlessly in a command-line environment, where user interaction is minimal. However, its outputs are readily visible throughout its execution.


Initialization Process

Upon its first execution, USBScanner automatically captures and stores information regarding all connected USB ports a nd devices. This initial setup phase occurs silently in the background, without necessitating any user input. It establishes a comprehensive reference list of known USB devices for subsequent comparisons.


Comparison with Known USB Devices

During subsequent runs, USBScanner compares the current state of connected USB devices with the reference list generated during initialization. It efficiently identifies any new or unknown devices that were not present during the initial scan.


Listing Unknown Devices

USBScanner produces a detailed output, listing all unknown USB devices detected during the comparison process. Each entry in the output includes essential information such as the Device ID, Name (if available), and Path/Bus details, enabling users to easily identify devices. This capability was born out of the need to identify unknown devices' details as a part of a much larger project I cannot discuss.


Automatic Execution and Output

As a standalone script, USBScanner operates autonomously, executing automatically without requiring any user interaction. Throughout its execution, the program provides clear and concise output, visible in the command-line interface in real-time. This allows users to monitor the process and review connected devices efficiently.



Project Outcome

Streamlined Task Efficiency

USBScanner transformed a time-consuming task, prone to human error, into a seamless process that now takes only seconds. By automating the identification of unknown USB devices and providing all necessary details swiftly.

EVE Links

Overview

Project Started May 2023
Project Ended June 2023
Platform Windows
Code C#


Functionality

Thorough Testing

Testing ensures stability and prevents crashes, crucial for maintaining responsiveness during critical moments. Crashes are particularly detrimental as they slow down response times when rapid action is required.


Redundancy and Fail-Safes

Multiple fail-safes and redundancy mechanisms are incorporated to enhance reliability and minimize the risk of failure, ensuring uninterrupted operation.


Space-Efficient Design

The program is designed to occupy minimal screen space on a secondary display, ensuring quick access to essential features without clutter. This streamlined design optimizes user workflow and efficiency.


Utilization of Unit Testing in C#

The program's development process includes unit testing in C#, a robust testing framework for C# applications. This systematic testing approach validates individual components and contributes to overall reliability.


Emphasis on Efficiency and Speed

The program prioritizes efficiency and speed in its design and functionality. While not necessarily the most memory-efficient, it leverages a garbage collector during memory-intensive tasks to maintain performance levels. This focus on efficiency ensures swift execution of tasks and enhances user productivity.



Interface Walkthrough

The main window of the application is divided into two scrollable displays, each presenting a list of entries containing descriptions linked to their corresponding URLs. These entries are sorted alphabetically, with the sorting operation executed upon saving and the sorted sequence preserved within the file.

In the settings section, users can access various functionalities. Firstly, they can instantly copy an entry to the clipboard by clicking on it. Additionally, if the "description is hyperlink" option is selected, the entry format changes to a clickable hyperlink, displaying both the description and the URL. For example, if the user selects this option and clicks on an entry titled "EVE Academy - How to get into incursions," the format would be EVE Academy - How to get into incursions. If this checkbox is not selected, the entry format displays the raw URL only, such as https://www.eveonline.com/eve-academy/careers/enforcer/incursions. Another feature in this section allows users to delete all data. Upon reloading the application, the file database will be populated with predefined entries.

In the details section, users can view comprehensive information about the selected entry, including both the link and description. This section also enables users to perform various actions such as editing, deletion, and addition of entries. Additionally, users can copy the entry to the lipboard with a single click, eliminating the need to select it twice. Furthermore, the application facilitates the opening of the link associated with the entry directly in the user's browser for further exploration.

The search function in the main window provides a dynamic experience by updating the WPF tables in real-time with each keystroke entered into the search field. The search is case insensitive, ensuring that matches are found regardless of the letter case used in the search query.

As the user types, the search function checks both the URL and description fields for matches. This comprehensive search ensures that relevant results are displayed to the user, whether they're searching for specific URLs or descriptions.

In addition to the search functionality, when an entry is clicked, the program provides additional information indicating whether it's from a third party or not. This information is crucial given the context in which the program is used. Even if the user enters something else in the search bar but doesn't click on a new entry, this feature ensures that they are informed about the source of the selected entry. This transparency helps users make informed decisions and maintain awareness of the origin of the displayed content, enhancing the overall usability and trustworthiness of the application.

To maintain responsiveness and reduce lag, asynchronous threading is employed. This means that the search operation is performed asynchronously in a separate thread, allowing the user interface to remain smooth and responsive even during intensive search operations. By leveraging async threading, the search function delivers a seamless user experience, providing quick and accurate results without sacrificing performance.

The add/edit entry window streamlines the process of adding or modifying entries in the database. Depending on whether the window is in "add entry" or "edit entry" mode, the behavior differs. In "add entry" mode, the fields are left empty, enabling users to input new data from scratch, while in "edit entry" mode, the fields are pre-filled with existing data from the database, facilitating modifications to specific entries.

Autofill is an option activated when the URL is from the game developer's affiliated websites, making data entry smoother by automatically populating fields based on keywords in the URL. If a YouTube link was provided, a web scraper only operates during the final database saving to confirm the channel's affiliation with the developer, resulting in a slight delay.

After validating and ensuring all information is complete, the system saves or updates the entry. The data is then sorted, and the updated data is saved to maintain database organization and integrity.

The delete prompt acts as a safety measure to prevent accidental deletions. It prompts users to confirm their deletion action, reducing the risk of unintentional data loss by providing an extra layer of defense against mistakes.

The "Visit link" prompt offers users the option to navigate to external websites. If users choose to proceed, the prompt effectively warns them before opening the link in a web browser, ensuring they are aware of the action. This preemptive warning prevents the surprise of a web browser page suddenly opening, providing a smoother and more controlled browsing experience for users.


Project Outcome

Enhanced Access with EVE Links

EVE Links aimed to streamline access to a plethora of valuable resources within the EVE Online universe. Offering a faster alternative to manually searching through notebooks for links, EVE Links provided quicker and easier access to essential information, improving efficiency in assisting fellow players and aiding in volunteering efforts within the game.


Efficient Keyword Search

With EVE Links' case-insensitive search function, users could input keywords to swiftly locate relevant information without the need for precise typing. This feature facilitated quicker assistance and information retrieval, empowering players to find what they needed promptly, whether it was ship fittings, market trends, or mission guides, thereby aiding in their volunteering tasks within the game.


Reliable Performance Under Load

EVE Links was engineered to maintain stability and responsiveness even under heavy usage or prolonged periods of inactivity followed by sudden intensive usage. This ensured that users could rely on the tool whenever they needed it, without worrying about slowdowns or interruptions to their volunteering efforts within the game.


Limited Adoption Despite Offer

Despite being offered to others, EVE Links was ultimately not adopted by the wider EVE Online community, who preferred to stick with their existing system. Despite the lack of wider adoption, EVE Links effectively served its intended purpose of providing quick access to valuable links and enhancing efforts within to help others in EVE Online.
